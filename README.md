# Flagify

Flagify is a modern, developer-first platform that combines **Feature Flags**, **Warehouse-Native Experimentation (A/B Testing)**, and **Product Analytics** into a unified, open-source system. 

It is designed to bridge the gap between engineering, product management, and data science teams by allowing you to control release risk, validate features, and measure business impact without complex pipelines.

---

## High-Level Architecture & How it Works

Flagify is architected to be incredibly fast, privacy-friendly, and secure. It runs on a "Warehouse-Native" model:

1. **Rule Evaluation (Local & Zero-Latency):** 
   You define feature rules, targets, and experiment weights in the Flagify dashboard. Flagify serves these configuration payloads via a light-weight API. The Flagify SDKs evaluate these rules locally on the client or server in under a millisecond, with zero network overhead per call and absolute privacy (your user attributes never leave your servers).
   
2. **Impression Tracking:**
   When a user is exposed to a feature flag or experiment, the Flagify SDK fires an "impression event" to your existing tracking system (e.g., Segment, Jitsu, RudderStack, Google Analytics, or custom event pipelines).

3. **Warehouse-Native Analysis:**
   Instead of forcing you to send your raw user events to a proprietary third-party analytics dashboard, Flagify queries your existing data warehouse directly. It writes read-only queries tailored to your schema, runs them on your database, and visualizes the results.

4. **Rich Visualization & Decision Making:**
   Flagify reads the compiled query results to generate detailed statistical dashboards, showing feature performance, confidence intervals, sample ratio mismatches (SRM), and custom metrics.

## Core Pillars & Features

### 🏁 Feature Flags & Targeting
* **Advanced Rule System:** Target users by any custom attribute (e.g., location, browser version, subscription tier, beta group).
* **Gradual Rollouts:** Safely roll out features to a percentage of users (e.g., 1%, 5%, 50%) to verify performance and system stability.
* **Remote Configuration:** Modify application behavior in real-time without redeploying code.
* **Mutually Exclusive Experiments:** Run multiple experiments concurrently without user overlap or statistical cross-contamination.

### ❄️ Warehouse-Native Experimentation
* **Supported Data Sources:** Direct connection to 11+ query engines, including:
  * Snowflake
  * Google BigQuery
  * ClickHouse
  * AWS Redshift
  * Databricks
  * Postgres / MySQL
  * Mixpanel
* **SQL Customization:** Customize the queries generated by Flagify to perfectly fit your data models.

### 🆎 Advanced Stats Engine
* **Statistical Methodologies:** Run experiments using either **Bayesian** or **Frequentist** statistics.
* **CUPED Variance Reduction:** Leverage CUPED to reduce noise and achieve statistical significance up to 50% faster.
* **Sequential Analysis:** Safely check experiment results early without risking false positives (peeking).
* **Sample Ratio Mismatch (SRM):** Automatic guards to detect and warn you if there is bias in user assignment.
* **Multi-Armed Bandits:** Automatically route traffic to the best-performing variation to optimize conversion during the run.

### 📊 Product Analytics Suite
* **Flexible Metrics:** Track simple conversion rates, continuous metrics (e.g., average cart value), ratios, or complex quantiles.
* **Dashboard Builder:** Build custom dashboards to monitor operational performance, user behavior, and product metrics side-by-side.
* **Screenshots & Documentation:** Attach context, screenshots, and markdown descriptions to metrics and experiments to build a historical repository of product knowledge.
